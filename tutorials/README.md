# Progressive component tutorials

These offline examples introduce one component at a time. Run commands from the repository root after `uv sync --dev --frozen`.

1. `basic_model` — canonical model invocation.
2. `tool_use` — validated environmental interaction.
3. `explicit_state` — inspectable state.
4. `planning` — bounded task decomposition.
5. `retained_context` — conversational context retained across turns.
6. `critique_validation` — deterministic checking and revision.
7. `bounded_tracing` — shared budgets and canonical events.

Then continue to [human approval](human_approval/README.md), which combines state, checkpointing and exact-action permission at a consequential boundary.

These pages teach **system components**. The [pattern examples](../patterns/README.md) next teach alternative execution flows; framework abstractions appear only after the common case study.
