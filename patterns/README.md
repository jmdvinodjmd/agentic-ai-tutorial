# Execution-flow patterns

Each example changes orchestration flow while reusing the same shared schemas, tools, budgets and traces.

The six groups are prompt chaining, routing and parallelisation, ReAct-style tool use, planner-executor, critic-reviser and orchestrator-worker. A pattern arranges existing system components; it does not introduce a new provider, state schema or framework.

Run each command from the repository root. After the patterns, continue to the [common research-assistant case study](../case_study/README.md).
